# Конструктор шаблонов электронной почты (Prototype)

**Индивидуальный проект** по паттерну **Prototype** (Прототип).

**Автор:** Солдатов Владимир Сергеевич

## Постановка задачи

Сделать консольное приложение, которое умеет:

- хранить шаблоны писем (тема, текст, список получателей);
- создавать письмо по выбранному шаблону;
- клонировать шаблон, менять его (тема/текст) и при этом не портить оригинал в реестре;
- сохранять готовое письмо в текстовый файл в папку `output/`.

Реализация: консольное приложение на **C# (.NET Framework 4.7.2)**.

## Реальный сценарий применения

Например, у небольшого сервиса постоянно повторяются письма:

- сброс пароля;
- уведомление о статусе заказа.

Если каждый раз копировать текст вручную - легко ошибиться и это занимает время.

Решение простое: один раз создаём шаблоны, а дальше берём шаблон и делаем письмо на его основе. Если надо поменять тему или текст - клонируем шаблон и редактируем клон.

## Паттерн Prototype (Прототип)

### Зачем он нужен

Шаблон письма - это объект с несколькими полями (тема, текст, список получателей). Часто нужен не новый шаблон “с нуля”, а копия существующего, но с небольшими изменениями.

Prototype позволяет **клонировать** объект и работать с копией.

### Какую проблему решает в проекте

| Проблема | Решение через Prototype |
|---------|--------------------------|
| Повторно создавать похожие шаблоны неудобно | Клонируем готовый шаблон |
| Изменения могут испортить оригинал | `Clone()` делает независимую копию |
| Нужны разные варианты одного шаблона | Делаем клон и меняем тему/текст |

### Ключевые элементы

- `IPrototype<T>` - интерфейс с методом `Clone()`
- `EmailTemplate` - конкретный прототип (шаблон письма)
- `TemplateRegistry` - хранит шаблоны и отдаёт клоны
- `EmailGenerator` - создаёт письма на основе клонированного шаблона

Клонирование сделано так, чтобы список получателей тоже копировался (глубокое копирование).

## Запуск

1. Открыть `Indibidual_Progekt.sln` в Visual Studio.
2. Выбрать стартовый проект **Indibidual_Progekt**.
3. Нажать **F5**.

## Использование

Меню в консоли:

```
=== EMAIL TEMPLATE CONSTRUCTOR ===
1. List templates
2. Create email message from template
3. Clone template and create message
4. Run demo scenarios
0. Exit
```

### 1) List templates

Показывает ключи шаблонов в реестре (например `password-reset`, `order-status`).

### 2) Create email message from template

Создаёт письмо по шаблону, просит ввести значения плейсхолдеров (`{name}`, `{code}`, `{orderNumber}`, `{status}`), сохраняет письмо в `output/<id>.txt`.

### 3) Clone template and create message (Prototype)

Клонирует выбранный шаблон, даёт изменить тему/текст клона, потом создаёт письмо и сохраняет в `output/`.
Оригинальный шаблон в реестре не меняется.

### 4) Run demo scenarios

Автоматически выполняет несколько сценариев и сохраняет результаты в `output/`.
